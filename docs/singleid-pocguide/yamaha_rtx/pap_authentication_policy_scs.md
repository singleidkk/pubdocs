# SCS評価制度におけるリモートアクセスVPN認証強化の考え方

文書更新日: 2026-05-16

参照資料: IPA「SCS評価制度 ★3・★4 要求事項・評価基準」（2026-05-16参照）

本書は、YAMAHA RTX のリモートアクセスVPNで SingleID のクラウドRADIUSを利用する場合に、SCS評価制度の観点でどのように説明できるかを整理した文書です。

SCS評価制度への適合を単独で保証するものではありませんが、リモートアクセスVPNの認証強化を説明する際の判断材料として利用することを目的としています。

## 結論

YAMAHA RTX と SingleID を組み合わせたリモートアクセスVPNの2要素認証は、SCS評価制度における認証強度、多要素認証、パスワード設定ルール、ネットワーク境界防護の考え方に沿った対策として説明できます。

SCS評価制度の公開要求事項では、現時点でリモートアクセスVPNそのものを名指しした要求は確認できません。ただし、★4では「インターネットを経由して社内環境へ接続する場合」に多要素認証を使用する要求が示されており、リモートアクセスVPNを説明しやすい項目になっています。

本構成では、パスワードに加えてワンタイムパスワード（OTP）を利用します。これにより、静的パスワードのみを利用するVPN認証と比較して、パスワード漏えい、使い回し、リスト型攻撃に対するリスクを低減できます。

## SCS評価制度との関係

本書では、IPAが公開している「SCS評価制度 ★3・★4 要求事項・評価基準」（2026-05-16参照）のうち、YAMAHA RTX のリモートアクセスVPN認証と関係する項目を整理します。

SCS評価制度の公開要求事項では、現時点でリモートアクセスVPNそのものを名指しした要求は確認できません。ただし、認証の強度、多要素認証、パスワード設定ルール、ネットワーク境界防護、ログ取得に関する項目は、リモートアクセスVPNの認証方式を評価する際の判断材料になります。

本構成は、SCS評価制度への適合を単独で保証するものではありません。SCS評価制度では、認証方式だけでなく、資産管理、ログ管理、ネットワーク境界防護、運用ルールなども含めて評価されます。そのため、本構成は「SCS評価制度の認証強度や多要素認証の考え方に沿った対策」として位置づけるのが適切です。

### SCS評価制度の観点での説明例

YAMAHA RTX と SingleID を組み合わせたリモートアクセスVPNの2要素認証は、SCS評価制度における認証強度、多要素認証、パスワード設定ルール、ネットワーク境界防護の考え方に沿った対策として説明できます。

特に、リモートアクセスVPNはインターネット経由で社内環境へ接続する入口になるため、静的パスワードのみで認証するのではなく、パスワードとOTPを組み合わせることにより、認証強度を高めることができます。

一方で、SCS評価制度は認証方式だけで評価されるものではありません。SCS評価制度の観点で説明する場合は、以下も合わせて確認してください。

- SingleID側のパスワードポリシー
- OTPの利用対象ユーザ
- YAMAHA RTX側のVPN設定と管理アクセス設定
- RADIUS共有シークレットの強度
- VPN装置とRADIUSサーバ間の通信経路制限
- 認証ログおよびVPN接続ログの取得・保管
- 管理者IDの管理手順

## ローカル認証DBとの比較

YAMAHA RTX のローカル認証DBで静的パスワードのみを利用する場合、RADIUS通信によるパスワード送信は発生しません。

一方で、静的パスワードのみの認証では、次のリスクが残ります。

- パスワードの使い回し
- 漏えい済みパスワードを使ったリスト型攻撃
- 弱いパスワードに対する総当たり攻撃
- 退職者や異動者のアカウント管理漏れ
- 拠点や機器ごとの認証情報管理のばらつき

SingleID のクラウドRADIUSを利用した2要素認証では、パスワードに加えてOTPを要求できます。そのため、パスワードが漏えいした場合でも、それだけでVPN接続が成立しにくくなります。

この点で、VPN全体の認証リスクを比較して判断することが重要です。

## 判断時のポイント

SCS評価制度の観点で確認する場合は、次の観点で整理すると判断しやすくなります。

- 認証情報がそのまま平文で送信されていないこと
- VPN通信が暗号化されていること
- RADIUS共有シークレットが十分に強固であること
- パスワードのみではなくOTPを組み合わせていること
- 管理者が利用者の追加、削除、無効化を適切に管理できること
- 認証ログを確認できること
- VPN装置やRADIUS設定の変更権限が適切に管理されていること

## 認証情報の保護に関する補足

SCS評価制度の観点では、多要素認証の有無、パスワードポリシー、ログ取得、ネットワーク境界防護を中心に整理するのが適切です。

一方で、社内確認の過程で「認証情報が平文で送信されるのか」が論点になる場合があります。その場合は、次のように説明できます。

YAMAHA RTX と SingleID を組み合わせたリモートアクセスVPNの2要素認証では、RADIUSのPAP認証を利用します。ただし、RADIUSにおけるPAPでは、パスワードはRADIUS共有シークレットとRequest Authenticatorを用いて暗号化されます。そのため、パスワードがそのまま平文で送信されるわけではありません。

なお、ここでいう暗号化は、RADIUSパケット全体ではなく、パスワード情報に対する暗号化です。通信全体をTLSで保護するRadSecとは異なります。
