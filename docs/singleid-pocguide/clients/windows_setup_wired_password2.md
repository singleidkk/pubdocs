# Windows: パスワードで有線LANアクセスの認証（詳細設定）

以下、NICに認証設定する方法です。

1. Windowsの**スタートメニュー＞アプリ、設定、ドキュメントの検索**より、**ncpa.cpl**を入力し、Enterキーを押下します。
2. **ネットワーク接続**画面が表示されるので、該当の有線認証用NICを右クリックし**プロパティ**を選択します。**イーサネットのプロパティ**画面が表示されます。
3. **認証**タブ画面で、以下を設定し、**設定**ボタンをクリックします。

    | **設定項目** | **設定内容** |
    | :--- | :--- |
    | **IEEE802.1X認証を有効にする** | チェックを入れます。 |
    | **ネットワークの認証方法の選択** | **Microsoft: 保護された EAP (PEAP)**を選択します。 |

4. **保護された EAP のプロパティ**画面で、以下情報を設定し、**構成**ボタンをクリックします。

    | **設定項目** | **設定内容** |
    | :--- | :--- |
    | **証明書を検証してサーバーのIDを検証する** | チェックします。 |
    | **次のサーバーに接続する** | チェックし、**SingleID 管理者ポータル＞認証＞RADIUS**画面の**基本情報**タブの**ホスト名**を入力します。 |
    | **信頼されたルート証明機関** | **ISRG Root X1**を選択します。 |
    | **認証方法を選択する** | **セキュリティで保護されたパスワード (EAP-MSCHAP v2)**を選択します。 |
    | **高速再接続を有効にする** | 必要に応じてチェックします。 |

    !!! info
        SingleIDのRADIUSサーバ証明書はLet's Encryptで発行されるため、短い周期で定期的に更新されます。**次のサーバーに接続する**にRADIUSサーバのホスト名を入力することで、証明書更新後も同じホスト名のサーバ証明書として検証できます。

5. **EAP MSCHAPv2 のプロパティ**画面で、**Windows のログオン名とパスワードを自動的に使う**のチェックを外し、**OK**ボタンをクリックします。

    !!! info
        このチェックを外すことで、Windowsのログオン情報ではなく、接続時に入力するSingleIDのユーザ名とパスワードで認証できます。

6. **保護された EAP のプロパティ**画面で、**OK**ボタンをクリックします。
7. **イーサネットのプロパティ**画面の**認証**タブに戻ります。**追加の設定**ボタンをクリックします。
8. **802.1Xの設定**タブで、以下情報を設定し、**OK**ボタンをクリックします。

    | **設定項目** | **設定内容** |
    | :--- | :--- |
    | **認証モードを指定する** | チェックします。 |
    | **認証モード** | **ユーザ認証**を選択します。 |

9. **イーサネットのプロパティ**画面で、**OK**ボタンをクリックします。**ネットワーク接続**画面を閉じます。
10. 認証情報の入力画面が表示された場合は、SingleIDの**ユーザ名**と**パスワード**を入力します。
11. 接続成功したことを確認します。
